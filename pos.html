The Challenge

I was asked to help lay the groundwork for a point of sales platform. I worked with the product owner closely to figure out the needed functions of the POS suite. This new POS system would work with our existing restaurant inventory systems, allowing the managers to better see what items are being ordered, the amount of stock each store needs to keep on hand, and hopefully minimize waste of overstocked, perishable items.

Getting Started

initial point of sales sketches

After a number of sessions with the product owner to discuss the functionality needed, I did a whole lot of research. I personally did not have any retail work experience using this kind of system, so this was all new to me (I did work at a movie theater in high school, but the setup there was quite low tech). Next, I began working on some visuals to start structuring the POS system and getting my head wrapped around how it would work.

First, I made a chart to iron basic functionality for the point of sales platform. We were able to clarify our MVP (Minimum Viable Product), as well as identify less necessary features and set those as stretch goals for later updates.

mvp chart

Since the POS program needs to work with our existing inventory management software, the product owner and I worked together with backend developers to better understand the data these programs will be sharing. I created a chart to show what information each function POS needed.

information chart

Can I take your order?

Once we had the groundwork completed, I started to create a flowchart to see the user flow of creating an order. While creating it, I got a better understanding of the difference between creating an eat in and take out order, as well as modifying existing orders.

wireflow diagram

Making wireframes of the order interface was my next step. Here I was able to implement all the knowledge I learned from my previous team discussions and research. It was very helpful to start seeing how the information would be inputted and displayed.

point of sales wireframes

Next Steps

I moved to a different team before this project completed the initial design stages. Wireframing of the manager tools and sales operation interfaces would have been the next stage in my design process.
